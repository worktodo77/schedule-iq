The published arithmetic is otherwise recognizable: population z-scores are
computed over each fundamental, available z-scores are averaged per update,
and the held-span mean is multiplied by the held-update count. The findings
below concern identity, temporal continuity, and evidence completeness, not an
inference of intent.

## Scope and current behavior

Section 11.2 and matrix row LI-12 define `DDI = held_updates * mean
deterioration z-score`, with a trailing run of at least two consecutive
updates, five named fundamentals (RDI, BWI, FCBI, near-target constraints, and
DUR-04 compressions), and NOT COMPUTABLE for fewer than three schedules or no
held span. `analytics/li_provocative.py::directed_date_index` (lines 447-544)
assembles the series; `_held_run` (428-444) selects the span and
`_ddi_arithmetic` (402-425) performs the published arithmetic.

The result remains `privileged=True`, `surface="internal"`, and carries the
triage sentence cap. All findings below therefore affect the citable internal
decomposition, not today's report-card grade.

## D1 - Target identity is still code-keyed (High) - CONFIRMED

`directed_date_index` imports `_find_by_code` and resolves the target finish
with `_find_by_code(s, tgt)` for every schedule (lines 460-473); the near-target
constraint count repeats the same lookup (around lines 493-495). The selected
milestone's persistent UID is never carried forward. This is directly contrary
to the shipped R-ID UID-first baseline and differs from the corrected BWI path.

**Probe.** Three schedules contain the same UID `u1`; the target is `T` in the
first file and is re-coded to `T_RECODE` in the next two. Passing the same
three-update index rows and `target="T"` returns:

```text
same UID across schedules: ['u1', 'u1', 'u1']
target_code = 'T'
held_updates = 1
ddi = None
reason = NOT COMPUTABLE: the milestone T did not hold station across two or
more consecutive updates (no directed-date span to measure)
```

The date did hold in all three records; only the code changed. This is an
implementation defect and an R-ID portability blocker. It has no grade impact
at weight 0, but it can erase the privileged DDI evidence for a real series.

**Ruling requested:** resolve the anchor once, retain its UID, match UID-first
with a narrow code fallback, and disclose the fallback. Add a re-code fixture
to the DDI/R-ID identity suite.

## D2 - A missing update is skipped rather than breaking consecutiveness
(Medium) - CONFIRMED

`_held_run` first filters out every `None` finish (`idx = [i ... if d is not
None]`) and then compares the remaining dates. Consequently, a missing target
in the middle of a series is not a break in the trailing run. Matrix LI-12
requires consecutive updates, not merely consecutive present observations.

**Probe.**

```text
_held_run([2025-01-01, None, 2025-01-01]) == [0, 2]
```

The returned two-index span crosses the update for which the target finish was
absent. With three schedules this can satisfy the `held_updates >= 2` guard and
produce a DDI as if the date had held across adjacent updates. This is a defect
(or an unstated spec gap) in the temporal evidence surface.

**Ruling requested:** treat a missing finish as a continuity break (or return
NOT COMPUTABLE for the affected span), and add a middle-update-missing probe.
If a policy intentionally bridges missing observations, disclose that policy
and rename the measure from consecutive updates.

## D3 - Fundamental coverage is not required or disclosed (Medium) - CONFIRMED

`_ddi_arithmetic` averages only the non-`None` z-scores at each update and
returns a numeric index whenever any fundamental remains. There is no minimum
fundamental coverage, per-update denominator, or result disclosure identifying
which named fundamentals were absent. The matrix names five fundamentals and
describes their mean, so a one-series result needs an explicit coverage rule.

**Probe.** RDI is the only populated fundamental; BWI is missing for all three
updates. The held span is the last two updates:

```text
ddi = 0.17677669529663687
held_updates = 2
mean_deterioration_z = 0.08838834764831843
BWI (bow-wave) = [None, None, None]
reason = ''
```

The result is computable and presented as DDI even though four of the five
named inputs are zero-variance or absent. This is primarily a methodology and
disclosure gap rather than an arithmetic error. It can materially change the
internal evidence while leaving the grade unchanged.

**Ruling requested:** choose and publish a minimum coverage/denominator rule
(or explicitly affirm available-only averaging), include per-fundamental
coverage and the effective count in `DdiResult`, and add sparse-input fixtures.

## Guardrail confirmation and lower-severity notes

- DDI is privileged/internal and weight 0 in scorecard `N17` / LI-12.
- The interpretation is capped at “increasingly unsupported by the schedule's
  own internal indicators” and leaves the reason for holding the date open.
- Population z-score arithmetic, zero-variance handling, and the short-series
  guard are internally consistent with the current matrix wording.
- The R-ID defect is localized to DDI's target/fundamental lookup; the existing
  BWI UID-first correction does not automatically protect DDI.

## Proposed tests once rulings land

- D1: same UID, changed code across three updates; DDI remains computable and
  reports the display code plus UID-first resolution.
- D2: missing middle target finish breaks the held run.
- D3: sparse fundamentals either return documented NOT COMPUTABLE or disclose
  the coverage denominator and available-only calculation.
